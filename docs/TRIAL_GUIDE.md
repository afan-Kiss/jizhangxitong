# 新手试用流程（明天真实试用前看这一篇）

试用模式下记的数据会带「试用」标签，**不会和正式账本混在一起**。试完可以一键清理，或确认无误后转正式。

## 第一步：扫镯子

1. 打开 App → 底部「镯子」
2. 输入或扫描镯子编号（如 F0007584）
3. 系统自动从扫码枪同步成本、图片

> 前提：本地 Worker 在线 + 扫码枪 API（7789）正常。可双击「一键检查状态.bat」确认。

## 第二步：记一笔支出

1. 底部「记支出」
2. 填金额、类型、报销人
3. 可关联刚同步的镯子编号

试用模式下，这笔会自动标记为「试用」。

## 第三步：上传付款截图

1. 在支出详情或创建页上传「付款截图」
2. 图片保存在本地电脑（省服务器空间）
3. Worker 在线才能上传成功

## 第四步：登记销售

1. 底部「销售」→ 登记销售
2. 选镯子、填销售金额
3. 系统自动算成本快照和毛利

试用销售同样带「试用」标签。

## 第五步：导出 Excel 报销

1. 「我的」→ 报销导出
2. 选日期范围 → 导出
3. Excel 内嵌付款截图，附图行不重复金额

试用模式下导出的表只含试用支出，不影响正式导出。

## 第六步：查看体检状态

双击 **「一键系统体检.bat」**，看：

- 服务端 / Worker / 扫码枪是否正常
- 备份是否过期
- 密钥、密码是否安全

---

## 试用模式怎么开关

**设置 → 我的 → 试用模式** 打开开关。

或管理员执行：`npm run trial:prep`（会自动检查并提示）

## 试用数据怎么清理

- 设置页点击「清理试用数据」
- 或：`npm run trial:cleanup`
- 或双击：`一键清理试用数据.bat`

清理逻辑：支出作废、销售退款，**不删本地图片文件**。

## 试用数据怎么转正式

试跑确认没问题后：

- 设置页 → 「试用转正式（全部）」
- 或 API：`POST /api/trial/promote` body `{ "all": true }`

会写操作日志，之后按正式数据统计和导出。

## 一键命令（不用复制，双击 bat 即可）

| 文件 | 作用 |
|------|------|

| 一键系统体检.bat | 全面健康检查 |
